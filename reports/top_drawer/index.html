Case 设计与观察

这些 case 不是同一种 OOD：有的是当前环境的官方任务，有的是同义改写，有的是自定义对象-目标组合， 也有的是把 LIBERO 其他 task id 的官方 instruction 放到当前固定 libero_goal/task_id=3 环境里执行。

整体观察

当前官方任务 两个模型都能完成原始 bowl -> top drawer 任务。
同义改写鲁棒性 pi0.5 在本次运行中完成了两条 paraphrase。MolmoAct2 完成了 “upper drawer” 这条改写，但没有完成 “top drawer of the cabinet” 这条表达。
对象 grounding 两个模型在部分对象替换上会被原始 bowl 行为吸收。最清楚的失败包括：两个模型的 plate drawer、MolmoAct2 的 wine bottle drawer、pi0.5 的 cream cheese drawer。
自定义组合成功例子 MolmoAct2 成功完成 cream cheese -> drawer。cream cheese 是 LIBERO 官方物体，但这个 drawer 组合不是 130 条官方 instruction 之一。
跨任务关系迁移 relation probe 不能简单叫全局 OOD：bowl -> plate、bowl -> stove、bowl -> cabinet 都是 LIBERO 其他 task id 里的官方 instruction。两个模型在本次运行中都完成了这三条关系迁移。
